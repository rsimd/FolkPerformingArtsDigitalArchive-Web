## 2. マーカー色の優先（推奨）

1. **緑**: `has_3d_motion` が真（かつ同意範囲内）
2. **水色**: `has_video` が真で 3D なし
3. **第4色**（例: 琥珀）: 個別詳細ページあり、動画・3D なし
4. **灰**: 個別詳細ページなし（スタブのみ等）

凡例テキストとアイコンを併用し、WCAG のコントラストを満たす。ヒーローは半透明スクラム・テキストシャドウ等で**オーバーレイ可読性**を確保する。

## 3. 情報設計と URL（既存 MVP との関係）

- **個別詳細**の URL は **`/performances/<slug>/`** 形式の**静的パス**を正とする（GitHub Pages のベースパスに合わせてプレフィックスを付けてもよい）。
- 現状の [templates/map.html](../../templates/map.html) に相当する**民俗芸能地図 MVP**は、次のいずれかで**トップと分離**する（実装で一つに決め、README に書く）:
  - **案 A**: サイトルート `/` をランディング＋俯瞰にし、既存 UI を **`/map`** または **`/explore`** に退避する
  - **案 B**: 静的ビルドで `index.html`（ランディング）と `explore.html`（既存相当）を分離する
- **検索への導線**: ヘッダまたはヒーロー下に「キーワードで探す」を置き、検索ページ（例: `/search.html`）へ遷移させる（[list-and-discovery.md](list-and-discovery.md)）。

## 4. 俯瞰地図のデータ結合とヒット領域

- 登録済みの**全エントリ**を地図に載せる。エントリは 1 つ以上の `Location`／**`boundary`**／代表点を持ちうる。
- **クリックのヒット領域**は実装で次のいずれかまたは併用する（併用時は**優先順位をドキュメント化**する）:
  - **(a)** 市町村ポリゴン（Natural Earth / 国土地理院系等の境界 GeoJSON を静的同梱し `feature-state` でハイライト）
  - **(b)** エントリの `boundary` ポリゴン（[map-and-geojson-contribution.md](map-and-geojson-contribution.md)、親文書 §4.2）
  - **(c)** 代表点のクラスタ
- **同一市町村内に複数エントリ**がある場合は **地域クリック → リスト** パターンを必須とする。1 件のみのときは直接詳細へ遷移する UX を許容する。

### 4.1 地域パネル用の集約データ

同一地域に紐づく **`performances[]`** をクライアントが `places.json` から集約してもよいし、ビルド時に **`data/by_region/<prefecture>_<municipality>.json`** を生成してもよい。**両方出す場合は正本を一つに固定**する（親文書 §4.6）。

パネルは**右パネルまたはモーダル**のカード一覧とし、各カードに詳細 URL・`summary`・**status**（`content_flags` と一致）を載せる。

### 4.2 灰マーカー（詳細ページなし）のカード

リンク先が**プレースホルダ**または「準備中」ページになる場合は、カード上にその旨を明示する。地図マーカー色（灰）と矛盾させない。

## 5. トップ用スライドショー写真

承認済みエントリの代表写真 URL を **`home_slideshow_manifest.json`**（ビルド時生成または手動キュレーション）に集約する。著作権・肖像の許諾が付いている写真のみを含める運用とする（親文書 §4.6）。
