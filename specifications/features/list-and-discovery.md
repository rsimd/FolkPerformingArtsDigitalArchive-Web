## 1. 概要

利用者が民俗芸能エントリを一覧し、キーワードや属性で絞り込み、個別芸能ページへ遷移するまでを扱う。情報アーキテクチャ上の `TopAndList` に相当する。トップの俯瞰地図・ヒーローは [home-and-overview-map.md](home-and-overview-map.md) を参照する。

## 2. 要件

| 要件 ID | 説明 |
|---------|------|
| F-LIST-01 | 民俗芸能エントリの一覧を表示する（カードまたは表形式） |
| F-LIST-02 | 検索: キーワード（芸能名、地名、団体名） |
| F-LIST-03 | フィルタ: 市町村、芸能種別タグ、文化財指定の有無・区分（参考表示） |
| F-LIST-04 | 一覧から個別ページへ遷移する |
| F-SEARCH-01 | ビルド時に `search_index.json` を生成し静的配信する |
| F-SEARCH-02 | 検索対象フィールドは親文書 §3.1 における F-SEARCH 表のとおり。長文全文は後段で拡張可 |
| F-SEARCH-03 | **インデックス禁止**: 寄稿者連絡先、管理者メモ、非公開フィールド、同意書の生データ。公開 JSON と**同等の範囲**のみを連結する |

## 4. 静的検索のランタイム（F-SEARCH）

- 検索ページに入ったときだけ `search_index.json` を **遅延 `fetch`** してもよい（初回 LCP 対策）
- 将来、インデックスが肥大したら **チャンク分割**を検討する（親文書 Open Questions 8・16）
- UI: 単一入力＋デバウンス付き即時結果、または送信ボタン。ハイライトは任意

## 5. 実装フェーズで詰める項目（追記用）

- UI パターン（カード / 表）の確定
- MiniSearch と Fuse.js の選定、日本語正規化（NFKC 等）
- ページネーションまたは無限スクロール
- 検索結果に `content_flags` を短縮表示するか（Phase 1 以降）
